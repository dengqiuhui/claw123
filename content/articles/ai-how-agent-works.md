## AI Agent 的核心循环

每个 AI Agent 都在不断重复一个**感知 → 思考 → 行动**的循环：

```
┌─────────────────────────────────────────┐
│                                         │
│   👀 感知 ──→ 🧠 思考 ──→ 🎬 行动     │
│      ↑                        │        │
│      └────────────────────────┘        │
│              反馈结果                    │
└─────────────────────────────────────────┘
```

---

## 第一步：感知（Perceive）

AI Agent 通过多种方式感知环境：

### 视觉感知
- **屏幕截图** — 看你的桌面在显示什么
- **浏览器内容** — 读取网页上的文字和按钮
- **文件预览** — 扫描文件内容

### 文本感知
- **IM 消息** — 理解你发的指令
- **邮件内容** — 读取收件箱中的邮件
- **文档内容** — 分析 Word、PDF、Markdown 等

### 状态感知
- **系统状态** — 当前打开了哪些应用
- **时间信息** — 现在是什么时间、星期几
- **上下文** — 你刚才做了什么操作

---

## 第二步：思考（Reason）

这是 AI Agent 最核心的环节。当 AI Agent 收到你的指令后：

### 任务拆解

```
你的指令：「帮我整理本周的邮件并写周报」
         ↓
      拆解为：
      1. 查看本周收件箱
      2. 按重要性分类
      3. 提取关键信息
      4. 汇总为周报格式
      5. 保存为文档
```

### 工具选择

AI Agent 会自行决定使用哪些工具：

| 任务 | 选择的工具 |
|------|-----------|
| 查邮件 | 邮件客户端 API |
| 提取信息 | 文本分析能力 |
| 写周报 | 文档生成技能 |
| 搜索资料 | 网页搜索技能 |

### 规划执行路径

AI Agent 会制定执行计划：
1. 先做什么
2. 中间检查什么
3. 最后输出什么
4. 如果失败怎么办

---

## 第三步：行动（Act）

行动环节 AI Agent 会**真正在电脑上操作**：

### 可以做的事
- 🖱️ 移动鼠标、点击按钮
- ⌨️ 输入文字、快捷键
- 📁 创建、移动、重命名文件
- 🌐 打开网页、填写表单
- 📧 发送邮件、回复消息
- 📄 创建和编辑文档

### 行动的限制
- ❌ 不能访问你未授权的文件
- ❌ 不会被允许的操作系统会拦截
- ❌ 重要操作会请求你的确认

---

## 完整案例：让小龙虾帮你订会议室

来看看一个完整的「感知 → 思考 → 行动」过程：

```
👀 感知：
收到你的指令：「帮我订明天下午3点的会议室，邀请张三和李四」
读取通讯录，知道张三和李四的邮箱
读取日历，确认你明天下午3点空闲

🧠 思考：
1. 打开会议室预订系统
2. 筛选明天下午3点可用的会议室
3. 预订最合适的那间
4. 自动发送邀请给张三和李四

🎬 行动：
1. 浏览器打开 meeting.company.com
2. 登录你的账号
3. 选择日期和时间
4. 找到空闲会议室并预订
5. 填写参会人邮箱
6. 发送会议邀请
7. 回复你：「已预订3号会议室，邀请已发送」
```

## 为什么 AI Agent 有时会"卡住"？

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 操作失败 | 网页结构变了 | 重试或手动操作 |
| 权限不足 | 未授权该操作 | 到设置中授权 |
| 任务太长 | LLM 上下文不够 | 把任务拆分发送 |
| 结果不准 | 指令不够清晰 | 写更具体的指令 |
